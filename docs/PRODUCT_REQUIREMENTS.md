# CaseIntelix Product Requirements Document (PRD)

## 1. Executive Summary
CaseIntelix is an enterprise-grade, multi-tenant legal case-intelligence platform. It allows law firms to securely upload case files, perform high-accuracy hybrid search, and ask questions across matters while receiving answers strictly grounded in the document text with page-level citations.

## 2. Multi-Tenancy & Data Isolation
* **Tenant Isolation**: Strict logical segregation of all data. All database tables must contain an `organization_id` linked to the tenant organization.
* **Matter-Level Access**: Matter members only. Users must be explicitly associated with a matter to read or write any information regarding that matter.
* **Audit Trails**: Immutable log of all reads, writes, and AI operations.

## 3. Document Processing Pipeline
* **Supported Formats**: PDF files up to 100MB.
* **Extraction Quality**: High-fidelity text extraction with page boundaries preserved. Quality metrics must trigger warnings or OCR fallback if characters are sparse or poorly formed.
* **Chunking**: Page-aware semantic chunking with overlap (target: 512 tokens with 64-token overlap). Chunks must never span page boundaries.

## 4. Search and RAG (Retrieval-Augmented Generation)
* **Hybrid Search**: Combination of keyword (PostgreSQL FTS) and vector similarity (pgvector).
* **Citations**: Every claim generated by the assistant must reference the exact document name, page number, chunk ID, and a matching text excerpt.
* **Abstention**: If search results do not contain the answer, the LLM must explicitly refuse/abstain rather than speculate or hallucinate.
* **Disclaimers**: Responses must display legal warnings indicating the output is a hypothesis requiring human review.
