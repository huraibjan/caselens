# Plan Phase Workflow

## Trigger
Before implementing any new feature or significant change.

## Steps
1. **Understand Requirements**: Review the relevant section of PRODUCT_REQUIREMENTS.md
2. **Check Architecture**: Verify alignment with SYSTEM_ARCHITECTURE.md
3. **Database Impact**: Assess schema changes needed (DATABASE_ERD.md)
4. **Security Review**: Check SECURITY_THREAT_MODEL.md for implications
5. **API Design**: Define endpoints per API_SPECIFICATION.md conventions
6. **Test Plan**: Outline test cases for the change
7. **Create Implementation Plan**: Document the approach in an artifact
8. **Get Approval**: Request user review before proceeding

## Outputs
- Implementation plan artifact with technical details
- Test plan
- Migration plan (if schema changes)
